## English

A tool for producing PDF listings of source code from your Git repositories. It shows the repository as a file tree with an estimated page count next to every file and a running total as you select. You pick files until the total matches the length you need, then export a paginated, print-ready PDF. Everything included is recorded in a database, and files already published are marked as used and locked out of future listings — so no fragment ever appears twice.

Built for recurring, document-oriented code exports where each batch must be a specific length and must not repeat earlier ones.

## Українською

Інструмент для генерації PDF-лістингів вихідного коду з Git-репозиторіїв. Показує репозиторій деревом файлів, де біля кожного файлу вказано, скільки сторінок він займе, і рахує загальну суму по мірі вибору. Обираєш файли, доки не набереш потрібний обсяг, і вивантажуєш посторінковий PDF, готовий до друку. Усе включене фіксується в базі, а вже опубліковані файли позначаються як використані й більше не потраплять у наступні лістинги — тому жоден фрагмент не з'явиться в документі двічі.

Створений для регулярних вивантажень коду в документи, де кожна пачка має бути заданого обсягу і не повторювати попередні.

## Po polsku

Narzędzie do generowania listingów kodu źródłowego w formacie PDF z repozytoriów Git. Pokazuje repozytorium jako drzewo plików, w którym przy każdym pliku widnieje szacowana liczba stron, a suma aktualizuje się na bieżąco w miarę zaznaczania. Wybierasz pliki, aż osiągniesz potrzebną objętość, i eksportujesz podzielony na strony PDF gotowy do druku. Wszystko, co trafiło do dokumentu, jest zapisywane w bazie, a opublikowane wcześniej pliki są oznaczane jako wykorzystane i nie pojawią się w kolejnych listingach — dzięki temu żaden fragment nie trafia do dokumentu dwa razy.

Stworzone z myślą o cyklicznych eksportach kodu do dokumentów, gdzie każda partia musi mieć określoną objętość i nie może powtarzać poprzednich.
